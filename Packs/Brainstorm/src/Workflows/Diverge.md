# Diverge Workflow

Running the **Diverge** workflow in the **Brainstorm** skill to expand the possibility space...

## Overview

Generate a wide range of genuinely different ideas. Quantity over quality. No judgment. The goal is to fill the table with options before evaluating any of them.

---

## Pre-Check

Before generating, understand the space:

1. **What's the actual problem/question?** (Restate in your own words)
2. **What constraints exist?** (Time, money, skills, dependencies)
3. **What's the user optimizing for?** (Speed? Quality? Learning? Fun?)
4. **What's already been tried/considered?** (Don't repeat known territory)

If any of these are unclear, ASK before diverging:
> "Before I brainstorm, help me understand: what matters most here — speed, quality, learning, or something else?"

---

## Generation Method

### Step 1: Conventional Ideas (3-4)
Start with the obvious, well-trodden approaches. Get them on the table so they're not "the elephant in the room."

### Step 2: Cross-Domain Analogies (2-3)
Ask: "What field has solved a similar problem differently?"
- How would a game designer approach this?
- How does biology solve this? (evolution, immune systems, neural networks)
- How would a 5-year-old describe the solution?
- What's the inverse/opposite approach?

### Step 3: Wild Card (1-2)
Use **BeCreative** (verbalized sampling) for at least one idea that breaks frame entirely:
- What if we didn't solve this at all? What would happen?
- What's the 10x version? (Not 10% better — 10x different)
- What would we do if we had unlimited resources? Now constrain back.

### Step 4: Combination Ideas (1-2)
Can any of the above be combined? X's simplicity + Y's power?

---

## Presentation Format

Present ideas as a numbered list. Keep each one SHORT — 1-3 sentences max. Don't over-explain. The user needs enough to react, not a full proposal.

```markdown
## Ideas

1. **[Short Name]** — [1-2 sentence description. What makes it different.]
2. **[Short Name]** — [Description]
3. **[Short Name]** — [Description]
...
7. **[Short Name]** — [Description] ← (this one's the wild card)

---

Which ones intrigue you? What's missing? Should I go deeper on any of these?
```

---

## Quality Checks

- [ ] At least 5 ideas (aim for 7-10 on complex topics)
- [ ] At least 3 are genuinely different approaches (not variations)
- [ ] At least 1 is "wild" — breaks assumptions or conventions
- [ ] None require paragraphs to explain (keep them punchy)
- [ ] You asked the user to react (don't just dump and wait)

---

## If the User Reacts

When the user responds with preferences:
- Note what they liked → These go into EXPLORE phase
- Note what they disliked → Understand WHY (the rejection reveals criteria)
- Note questions they asked → These are the interesting threads
- Transition to `Workflows/Explore.md` for the promising ones
